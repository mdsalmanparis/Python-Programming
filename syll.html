How This Syllabus Works

This roadmap has two tracks that run in parallel once you hit the DSA gateway. Phase 0 + Phase 1 must be completed fully first, followed by passing the Gateway checklist. Once passed, DSA runs as a daily side track (1 hr/day) while you continue Phase 2–6 as your main curriculum track.

Phase 0

Absolute Beginner Foundation

CORE ~3 weeks

Your true starting point. Nothing here is optional. Do not skip, do not skim.

0.1 Setup & First Program

  • Installing Python 3.11+, verifying with python --version
  • VS Code setup — Python extension, select interpreter, run with F5 or terminal
  • REPL — experimenting line by line with python in terminal
  • First script — print("Hello, World!"), save as .py, run with python filename.py
  • Comments — # single-line, """...""" multi-line docstrings
  • print() — multiple arguments, sep=, end=, f-strings preview
  • input() — reading user input, always returns a string
  • Indentation — 4 spaces standard, what IndentationError means

0.2 Variables & Basic Data Types

  • Variables — naming rules, snake_case, multiple assignment, swapping values
  • int, float, str, bool, None — what each one is
  • Floating point imprecision — 0.1 + 0.2 != 0.3 and why
  • type() and isinstance() — checking and validating types
  • Type conversion — int(), float(), str(), bool()
  • Operators — arithmetic, comparison, logical, short-circuit evaluation, augmented assignment

0.3 Strings in Depth

  • String creation, indexing s[0], slicing s[start:stop:step], reversing with s[::-1]
  • f-strings — f"Hello, {name}!" and string repetition "ha" * 3
  • String concatenation inefficiency in loops — use join() instead
  • Methods: .upper(), .lower(), .strip(), .replace(), .split(), .isdigit()
  • String immutability — cannot change a character in-place

0.4 Lists

  • Creating, mutability, indexing, and slicing
  • Adding elements — .append() O(1), .insert() O(n), .extend()
  • Removing elements — .remove(), .pop(), del, .clear()
  • Sorting — .sort() in-place vs sorted() returns new list
  • Shallow copy vs reference copy: why b = a does not copy lists
  • Nested lists and unpacking arrays

0.5 Tuples, Sets & Dictionaries

  • Tuples: Creating, immutability, fixed collections, returning multiple values, unpacking
  • Sets: Unordered, no duplicates, O(1) membership testing, union |, intersection &
  • Dictionaries: Key-value stores, accessing safely with .get(), .items() loops, ordering

0.6 Conditional Statements

  • if / elif / else — first True condition wins
  • Truthiness and falsiness — 0, "", [], {}, None are all falsy
  • Ternary expression — value = "yes" if condition else "no"

0.7 Loops

  • for loops — iterating over ranges, lists, dicts; enumerate(), zip()
  • while loops — conditional loop blocks, infinite loops, exit flags
  • Loop control — break, continue, else blocks on loops

0.8 Functions — Basics

  • def, return, default parameter values, keyword arguments, local vs global scope

0.9 to 0.12 Foundations

  • Built-in functions: min, max, sum, any, all, type casting
  • File I/O Basics: with open("file.txt") as f:, file modes, encoding rules
  • Error Handling: try/except/finally, custom checks, raising exceptions
  • Modules: math, random, os, sys, datetime, time, pip install
Phase 1

Python Core Depth

CORE HIRE ~4 weeks

The filter layer every backend interview starts with. No gaps, no hand-waving.

1.1 Mutability, Identity & Memory

  • id(), is vs ==, identity vs equality
  • Mutability rules: mutable vs immutable collections
  • Mutable default argument trap: def f(x=[]): bug and None sentinels
  • Shallow copy vs deep copy: copy.copy() vs copy.deepcopy()

1.2 Comprehensions & Advanced Iterables

  • List, dict, and set comprehensions; nested generators
  • Generator expressions: memory efficiency and lazy evaluation
  • Walrus operator := inside comprehensions and loops

1.3 Function Scopes & Closures

  • Positional, keyword-only (*), and positional-only (/) arguments
  • *args and **kwargs packing and unpacking
  • First-class functions, higher-order functions, closures, nonlocal keyword
  • Late binding gotcha in loops and functools.partial()

1.4 OOP & Magic Methods

  • Instance vs class vs static methods; properties and setters
  • Dunder methods: __init__, __repr__, __eq__, __hash__, __len__, __getitem__
  • Inheritance MRO (C3 linearization), diamond problem, abstract base classes
  • Dataclasses: @dataclass, field definitions, post-init hooks

1.5 to 1.8 Iterators, Decorators & Types

  • Iterator protocol: __iter__, __next__, StopIteration
  • Decorators: Function wrapping, metadata retention, class decorators, LRU cache
  • Type hints: Annotations, Generic classes, Protocol structural typing, mypy compiler
Checkpoint

🚦 DSA Gateway Checklist

Check these items off to confirm you are ready to begin the daily DSA side track.

From Phase 0
You can write a Python function from scratch with parameters, return values, and a docstring.
You understand list indexing, slicing, and the difference between .sort() and sorted().
You can use dict, set, and list naturally — adding, removing, iterating.
You understand for loops, while loops, break, and continue.
From Phase 1
You understand is vs == and can explain the mutable default argument trap.
You can write a list comprehension, dict comprehension, and generator expression.
You understand closures, *args, and **kwargs.
You can write a class with __repr__ and understand multiple inheritance MRO.
You can write a custom decorator using functools.wraps.
DSA Specific Python Libraries
You can use collections.defaultdict and collections.Counter.
You can use collections.deque for O(1) appends and pops.
You can use heapq and understand the max-heap negative number trick.
You can use bisect.bisect_left() and bisect.bisect_right() for binary searches.
Phase 2

Standard Library Mastery

HIRE JOB ~3 weeks

2.1 Collections & Paths

  • Counter operations, defaultdict factories, deque circular buffers, namedtuples
  • pathlib.Path: file system globbing, dir iterations, reading/writing text, mkdirs
  • os environments, subprocess, file manipulation via shutil

2.2 Formats & Utility Packages

  • JSON serialization: loading, dumping, custom encoders for unmappable classes
  • CSV parsing: reader, writer, DictReader, DictWriter
  • datetime timezone awareness, UTC, ISO format codes, parsing date strings
  • Regular expressions: re.search, re.match, capture groups, sub, split
  • logging log hierarchies, custom log configurations, log propagation controls
Phase 3

Concurrency & Async

HIRE ~3 weeks

FastAPI is async-first. This phase is non-negotiable.

3.1 Threading & Multiprocessing

  • CPython GIL: what it protects, its implications on I/O vs CPU bound programs
  • Threads: creation, Locks, RLocks, events, race conditions
  • Processes: Pool executors, multiprocessing module, Windows/macOS startup main block guard

3.2 asyncio

  • Event loop structure, coroutine invocation via async def and await
  • Running tasks concurrently: asyncio.gather, asyncio.wait, TaskGroups (Python 3.11+)
  • Async components: Locks, Semaphores, Queues, Context Managers, Iterators
  • Httpx async client sessions: requests, timeout options, connection pooling
Phase 4

Testing & Code Quality

HIRE ~2 weeks

4.1 pytest

  • Test discovery, assertions rewriting, fixture scopes, conftest.py
  • Parametrization, test markers, temporary directories, monkeypatch mockings
  • Exception checking, pytest-asyncio, ASGI test clients

4.2 Mocking & Code Linters

  • Mock objects, MagicMock, patch decorators, patch context managers, side effects
  • Mocking time, databases, third-party clients, file-readers
  • Ruff formatting/linting, mypy strict type checkings, pre-commit git hooks
Phase 5

Data & Database Layer

HIRE ~3 weeks

5.1 SQLAlchemy ORM & Alembic

  • SQLAlchemy 2.0 DeclarativeBase, Mapped columns, relationship keys, foreign keys
  • Sessions: Sessionmaker, AsyncSession transactions, raw select queries, joins
  • Eager loading: N+1 queries, selectinload, joinedload, lazy raise
  • Alembic migrations: autogenerations, upgrades, downgrades, stamp heads

5.2 Pydantic, Redis & Postgres

  • Pydantic schemas: field validators, computed fields, serialization mappings
  • Redis cache: Key-value caching, TTL configurations, list queues, hash representations
  • Postgres database: Indexes, composite keys, EXPLAIN plans, transaction isolation levels
Phase 6

Packaging, Tooling & DevOps

HIRE ~2 weeks

6.1 Virtualenvs, Docker & CI/CD

  • Virtual environments, pyproject.toml dependencies, fast package managers like uv
  • Docker containerizations: Dockerfiles, layer cache optimization, docker-compose orchestration
  • GitHub actions workflows, running tests in CI, pipeline secrets
  • 12-Factor app architectures: configurations via environment variables
Side Track

⚡ DSA — Parallel Track

HIRE 1 hr / day

DSA Subjects Schedule

  • Arrays & Strings: Two pointer, sliding windows, prefix sums, Kadane's maximum subarray
  • Hash Maps & Sets: Membership lookups, frequency maps, group anagrams
  • Stacks & Queues: deques, monotonic stack, min stacks, heaps priority queues
  • Linked Lists: Node linkages, Floyd cycle detections, reverses, LRU cache
  • Binary Search: Bisect indexes, search bounds, answer space searchings
  • Trees: Traversals (BFS/DFS), height calculations, LCA, Tri-prefix trees
  • Recursion & Backtracking: Call stack, backtracking recursion state branches, N-Queens
  • Graphs: Adj lists, BFS paths, cycle checks, topological sorting, Dijkstra
  • Dynamic Programming: Optimal substructures, memoizations, 1D/2D patterns
  • Bitwise Manipulations: Bit operations, bit count popcounts, bitmask permutations
  • Sorting algorithms: Merge-sort, Quick-sort, Cyclic sort
